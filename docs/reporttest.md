# Unittest
## Цели и задачи тестирования:
Проверка правильности вычисления функций, содержащихся в библиотеке geometric_lib.
## Описание трестируемого продукта:
В библиотеке geometric_lib находятся 4 файла, в каждом есть функции area и perimeter, которые вычисляют площадь и периметр для каждой фигуры (круг, прямоугольник, квадрат, треугольник).
## Область тестирования:
#### 1. Circle 
В файле содержится 2 функции: 
* `area(r)` - принимает радиус (int/float), возвращает площадь круга
* `perimeter(r)` - принимает радиус (int/float), возвращает периметер круга
#### 2. Rectangle
В файле содержится 2 функции: 
* `area(r)` - принимает длину и ширину (int/float), возвращает площадь прямоугольника
* `perimeter(r)` - принимает длину и ширину (int/float), возвращает периметр прямоугольника
#### 3. Square
В файле содержится 2 функции: 
* `area(r)` - принимает сторону (int/float), возвращает площадь квадрата
* `perimeter(r)` - принимает сторону (int/float), возвращает периметр квадрата
#### 4. Triangle
В файле содержится 2 функции: 
* `area(r)` - принимает сторону и высоту (int/float), возвращает площадь треугольника
* `perimeter(r)` - принимает стороны (int/float), возвращает периметр треугольника
## Стратегия тестирования
- Тестирование проводится с помощью библиотеки *unittest*, она помогает тестировать функции. Для тестирования функций создаются классы, например класс для круга: 
```
class CircleTestCase(unittest.TestCase):
```
Внутри () находится `unittest.TestCase` - используемый для создания новых тестовых примеров. Имена всех функций включают в себя test. Например функция проверки целых чисел для круга:
```
def test_int(self):
        self.assertEqual(area(1),pi*1*1)
        self.assertEqual(area(60),pi*60*60)
        self.assertEqual(perimeter(10),2*pi*10)
        self.assertEqual(perimeter(76),2*pi*76)
```
`assertEqual` используем, чтобы проверить, возвращают ли функции ожидаемое значение. То есть, assertEqual(a, b) проверяет a == b, и дальше идет проверка истина/ложь.
* Тип тестрования: функциональное модульное тестирование
## Критерии приемки:
После запуска теста должен быть ответ `OK`, если все сценарии учтены, если нет, то ответ будет `FAILED`. Сценарий - данные, которые будет вводить пользователь в программу.
## Ожидаемые результаты:
Все функции прошли тесты, где вводились/ось целое/ые или вещественные/ое числа/о, но в функциях не учтены случаи, когда пользователь вводит отрицательные/ое числа/о или символ/ы. Соответсвенно в таком случае, нет сообщения об ошибке.
